Ограничения:

  • форматы входных данных, сгенеренных программой для тестирования

Основная идея это конечно предсказание output на основе input (environment, arguments etc.) Нейронная сеть не может предсказать связь output -> input  для любой программы (какие тут ограничения (?)), то есть не может для любого датасета, то есть для одного набора данных input в output может быть преобразован, для другого - нет. Тест скрипт может быть сгенерен только там, где это преобразование можут быть предсказанно.

  • Посредством типа данных это ограничивает набор программ. Но опять же нам не нужны нейросети везде (однако есть преимущества в универсальном решении)
  • У каждой DSL комманды есть environment и предыдущая программа. Label является обязательной частью environment. Если лейбл дефолтный, является ли выбор следующей команды инваривантым? возможно бел лейбла и проблема вбыора быссмысленна
