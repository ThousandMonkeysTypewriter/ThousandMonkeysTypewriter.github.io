How It Works

1

Request a Review

Ready for a review? Connect PullRequest with your repository hosting service and request a review from the PullRequest dashboard. You can also tag the pull request by adding [pr] to the title.

2

Automation Tools Run

We run concurrently optimized static analysis and linter engines, all customized for your team's needs. We'll even try to set defaults based on your previous pull request histories.

3

Professional Code Review

A PullRequest reviewer or team is assigned to your pull request. They look through the code changes and automation outputs. Based on this info, they'll submit a review with suggestions, comments, and fixes directly in your existing workflow.

4

Changes Approved / Discussed

Have questions? Confused about one of our proposals? Not a problem! Comment right in GitHub, Bitbucket, Slack or our web application. Once everything looks hunky-dory, you push away using your normal CI/CD process.

5

Happy Code, Happy Product

We'll help you move faster and catch more issues. This leads to a happier team, a happier product, and makes us happy too!
